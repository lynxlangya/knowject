## 用户角色

- 独立开发者 / 项目所有者
- 全栈工程师，熟悉 React、Node.js、Python

## 技术背景

- 深度 Go expertise（参考 AGENTS.md）
- 新手 React（相对）
- 项目使用 TypeScript

## 协作偏好

### 提交信息
- 用户要求提交时，默认提供草案而非直接执行
- 需要包含标题 + Why/What/Validation/Risk 四段正文
- 多个单一目的改动优先拆分

### 工作流程
- 用户直接用 `/clear` 清屏，不依赖自动清理
- 使用 Claude Code 进行代码开发和调试
- 偏好中文沟通

### 反馈记录
- 喜欢简洁的响应，不需要每次都总结
- 确认成功后直接告知，不需要额外总结
