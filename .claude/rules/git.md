# Git Rules

## 提交信息约定

用户要求提交时，默认提供草案而非直接执行 `git commit`。

提交信息须包含：
- **标题**：简洁描述改动
- **Why**：改动的动机
- **What**：具体做了什么
- **Validation**：如何验证改动
- **Risk**：潜在风险或回滚点

多个单一目的改动优先拆分为多个 commit 草案，并说明推荐的提交顺序。

## 提交前检查

若存在未完成验证、既有阻塞或潜在回滚点，必须在提交草案中如实说明。

除非用户明确要求直接提交，否则默认只生成草案，不执行 `git commit`。

## 禁止操作

- 禁止在未确认的情况下 force push 到 main/master
- 禁止跳过 pre-commit hooks
